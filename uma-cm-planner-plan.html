§3 Guiding principles

Standing rules for every implementation session — mirrored into CLAUDE.md.

P1

Reuse first

5+ years of community tooling exists. Import > borrow data > port with attribution > build from scratch. Keep the Resource Map alive.

P2

Local-first, zero backend

Static site, data baked at build time, user data in IndexedDB, one-click JSON export/import. No server-side anything.

P3

Honest numbers

Real numbers where calculable; reliability tiers + evidence where it's RNG. Sims are estimations, not verdicts — surface caveats.

P4

Server-versioned data

Every record carries server + dataVersion. JP-ahead content is preview-only, never silently mixed into Global math.

P5

Hand-patchable data

Every generated dataset has a sibling *_overrides.json merged last, maintained by hand. Never edit generated files.

P6

Pure-function core

All mechanics live in src/core/ as pure, unit-tested TypeScript validated against community references. UI is thin.

Image assets — sourcing decision Decided · 2026-06-13

The UI is image-based (GameTora-style) for fast at-a-glance use mid-run. Bundle a curated, Global-only icon subset — not a hotlink, not the 415MB wholesale dump. A build step copies ~360 files (56 skill icons + 220 support_card_s chips + 84 uma portraits) from the local uma-tools/icons dump, converts to WebP (~5MB total) under public/data/icons/ with an icon-manifest.json. P2 (local-first/offline) forbids a hotlink primary; bundling is also lowest legal-risk under a NOTICE excluding all Cygames assets from the GPL grant. Resolution is deterministic & id-keyed (skill → iconId, card → cardId, uma → umaId, alt-outfits fall back to the base portrait). Community CDN hotlinking stays a lazy fallback only.
